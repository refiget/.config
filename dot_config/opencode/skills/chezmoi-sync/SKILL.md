## What I do
- Detect changed files via `chezmoi status`.
- Re-add each changed target file with `chezmoi re-add`.
- Auto-generate a commit message from staged changes.
- Commit updates in the chezmoi source repo.
- Apply final changes to the target files with `chezmoi apply`.

## When to use me
Use this skill after editing files managed by chezmoi and you want a consistent sync flow:
`re-add -> commit -> apply`.

## Workflow
1. Run `chezmoi status` to collect changed paths.
   - Output format: `<status> <target-path>` (e.g. `M .config/zsh/.zshrc`)
   - Status codes: `M` = modified, `A` = added, `D` = deleted
   - Ignore `D` (deleted) entries — no re-add needed.
2. For each `M` or `A` entry, run:
   - `chezmoi re-add "$HOME/<target-path>"`
3. Stage all changes in the source repo (do NOT use `chezmoi cd` — it opens a subshell):
   - `git -C "$(chezmoi source-path)" add -A`
4. Check if there is anything staged:
   - `git -C "$(chezmoi source-path)" diff --cached --quiet` → if exit 0, skip commit and report "No staged changes to commit".
5. Generate commit message from staged diff:
   - Run `git -C "$(chezmoi source-path)" diff --cached` to inspect changes.
   - Prefer concise, intent-based message style used in repo history.
   - Use this priority:
     - If one clear theme: `update <area> for <intent>`
     - If bug fix: `fix <area> <problem>`
     - If mixed changes: `update dotfiles`
   - Keep message lowercase, short, no trailing period.
6. Commit and apply:
   - `git -C "$(chezmoi source-path)" commit -m "<auto-generated-message>"`
   - `chezmoi apply`

## Guardrails
- Do not commit secrets (e.g. `.env`, credential files, private keys).
- Never use `chezmoi cd` in the workflow — it opens an interactive subshell and blocks execution.
- Always use `git -C "$(chezmoi source-path)"` to run git commands in the source repo.
- Derive commit message from staged changes only (`git diff --cached`), not from unstaged noise.
- Skip commit entirely if nothing is staged.

## Example
```bash
# 1. Check status
chezmoi status
# M .config/zsh/conf.d/20_aliases_core.conf

# 2. Re-add changed file
chezmoi re-add ~/.config/zsh/conf.d/20_aliases_core.conf

# 3. Stage
git -C "$(chezmoi source-path)" add -A

# 4. Commit
git -C "$(chezmoi source-path)" commit -m "update zsh aliases for reload shortcut"

# 5. Apply
chezmoi apply
```

## Auto message examples
- `update zsh aliases for reload shortcut`
- `fix zsh alias conflict`
- `update opencode config for chezmoi sync workflow`
- `update dotfiles`
