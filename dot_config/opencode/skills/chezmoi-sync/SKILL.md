## What I do
- Parse changed files from `chezmoi diff`.
- Re-add each changed target file with `chezmoi re-add`.
- Auto-generate a commit message from staged changes.
- Commit updates in the chezmoi source repo.
- Apply final changes to the target files with `chezmoi apply`.

## When to use me
Use this skill after editing files managed by chezmoi and you want a consistent sync flow:
`re-add -> commit -> apply`.

## Workflow
1. Run `chezmoi diff` and collect changed paths from lines like `+++ b/.config/...`.
2. For each changed path, run:
   - `chezmoi re-add "$HOME/<path-from-diff>"`
3. Enter source repo and stage changes:
   - `chezmoi cd`
   - `git add -A`
4. Generate commit message automatically from staged diff:
   - Prefer concise, intent-based message style used in repo history.
   - Use this priority:
     - If one clear theme: `update <area> for <intent>`
     - If bug fix: `fix <area> <problem>`
     - If mixed changes: `update dotfiles`
   - Keep message lowercase, short, and no trailing period.
5. Commit and apply:
   - `git commit -m "<auto-generated-message>"` (skip commit if nothing staged)
   - `chezmoi apply`

## Guardrails
- Do not commit secrets (for example `.env`, credential files, private keys).
- If `git diff --cached --quiet` is true, skip commit and report "No staged changes to commit".
- Keep commit messages short and intent-focused.
- Derive message from staged changes only (`git diff --cached`), not from unstaged noise.

## Example
```bash
chezmoi re-add ~/.config/zsh/conf.d/20_aliases_core.conf
chezmoi cd
git add dot_config/zsh/conf.d/20_aliases_core.conf
git commit -m "update zsh aliases for reload shortcut"
chezmoi apply
```

## Auto message examples
- `update zsh aliases for reload shortcut`
- `fix zsh alias conflict`
- `update chezmoi managed zsh config`
